# Экраны

Значит сейчас только три экрана, возможно будут добавляться по мере необходимости:

1. Auth
2. Feed
3. PostDetails


## Auth

Отвечает за авторизацию пользователя

### Вход

Мы начинаем с экрана авторизации при запуске

### Выход

- После успешной авторизация мы переходим в Feed

### Основные сценарии

#### 1. Успешная авторизация

1. Экран загружается
2. Пользователь вводит логин и пароль
3. Нажимает на кнопку авторизоваться
4. Пользователю открывается Feed

#### 2. Ошибка авторизации

1. Пользователь вводит неверные данные
2. Нажимает на кнопку авторизоваться
3. Пользователю показывается ошибка


## Feed

Собственна, лента (бесконечная?) с постами, где есть счётчик лайков и комментариев

### Вход

Из авторизации

### Выход

- Открытие поста
- Выход из аккаунта (обратно к экрану авторизации)

### Основные сценарии

#### 1. Загрузка ленты

1. Экран загружается
2. Пользователь видит посты
3. Пользователь может пролистать посты

#### 2. Лайк поста (и, соотвветсвенно, отмена лайка)

1. Пользователь нажимает на реакцию
2. Пользователь видит обновления счётчика
3. В случае ошибки возвращаем обратно

#### 3. Открытие поста

1. Пользователь нажимает на пост
2. Пользователю открывается PostDetails


## PostDetails

Тут мы собственно можно посмотреть пост целиком, посмотреть и оставить комментарии.  
(Динамически обновлять комментарии в фоновом режиме?)

### Вход

Из экрана ленты (добавить поддержку ссылок?)

### Выход

- Закрытие поста

### Основные сценарии

#### 1. Загрузка данных

1. Экран загружается
2. Пользователь видит пост
3. Пользователь может пролистать комментарии к посту

#### 2. Добавление комментария

1. Пользователь пишет свой комментарий
2. Нажимает отправить
3. При успехе комментарий добавляется к остальным комментариям
4. При ошибке отображается ошибка (проверки на сервер?)

#### 3. Лайк поста (и, соотвветсвенно, отмена лайка)

1. Пользователь нажимает на реакцию
2. Пользователь видит обновления счётчика
3. В случае ошибки возвращаем обратно
